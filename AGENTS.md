# apod-nasa — Agent Instructions

Minimal Flask web app that fetches and displays the [NASA APOD API](https://api.nasa.gov/). Deployed to Kubernetes via Kustomize with an nginx sidecar.

## Key Facts

- **Single-file app**: all logic lives in [`apod.py`](apod.py); templates in [`templates/index.html`](templates/index.html)
- Python ≥ 3.14, managed with **Poetry 2.x**
- Served by **uWSGI** in production (Docker); Flask dev server for local runs
- Exposed at `http://localhost:8000` locally, `https://apod.anetwork.cloud` in production

## Common Commands

```sh
poetry install                              # install all dependencies

export API_KEY=your_nasa_api_key            # optional, defaults to DEMO_KEY
poetry run python apod.py                   # run dev server on :8000

# Linting / formatting (CI mirrors these)
poetry run black .
flake8 --extend-ignore=E501 $(git ls-files '*.py')
pylint --disable=C0301 $(git ls-files '*.py')

# Docker
docker build -t apod-nasa .
docker run -e API_KEY=<key> -p 8000:8000 apod-nasa
```

## Configuration

All settings are environment variables:

| Variable  | Default      | Description                       |
|-----------|--------------|-----------------------------------|
| `API_KEY` | `DEMO_KEY`   | NASA API key                      |
| `PORT`    | `8000`       | Flask/uWSGI listen port           |
| `DEBUG`   | `False`      | Flask debug mode                  |
| `WORKERS` | `2`          | uWSGI worker count (Docker only)  |

## Architecture

```
[Ingress / HTTPRoute]
       ↓
  nginx:80  (sidecar, proxies to 127.0.0.1:8000)
       ↓
  Flask/uWSGI:8000
```

- `nginx` handles TLS termination forwarding and the `/healthz` liveness/readiness probe
- Flask exposes `/flask-healthz` for its own liveness probe
- `deploy/default.conf` is mounted via ConfigMap (`nginx-default-conf`), generated by Kustomize
- `deploy/httproute.yml` uses **Gateway API** (Envoy Gateway, namespace `envoy-gateway-system`)
- `deploy/ingress.yml` is a legacy Traefik Ingress (kept alongside HTTPRoute)

## Kubernetes Deploy

Manifests live in [`deploy/`](deploy/). Apply with Kustomize:

```sh
kubectl apply -k deploy/
```

Resources: `deployment.yml`, `service.yml`, `httproute.yml`, `ingress.yml`.  
`kustomization.yml` generates the `nginx-default-conf` ConfigMap from `default.conf`.

Image: `ghcr.io/ayresfonseca/apod-nasa:<tag>` — published automatically on GitHub Release via [`.github/workflows/publishing-image.yml`](.github/workflows/publishing-image.yml).  
Pull secret: `ghcr-apod` (must exist in the target namespace).

## Conventions

- Long lines (>79 chars) are intentionally allowed — `E501` and `C0301` are suppressed
- `requirements.txt` is NOT used; Poetry manages all deps
- `pyproject.toml` uses PEP 621 `[project]` table alongside `[tool.poetry]`
- Pre-commit hooks enforce: trailing whitespace, EOF newline, large-file guard, `poetry check`, `poetry lock`
- Do not add `requirements.txt` or use `pip install` directly
